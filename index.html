Lem-In

Pathfinding Optimisation Golang

Vue d'ensemble

Lem-In est un projet d'optimisation algorithmique dont l'objectif est de faire traverser une colonie de fourmis à travers un réseau de salles le plus rapidement possible. Le logiciel reçoit en entrée un fichier texte décrivant le nombre de fourmis et l'agencement de la fourmilière. Après avoir validé l'intégrité des données, le programme calcule le cheminement optimal pour acheminer l'ensemble des fourmis vers la sortie, en respectant la contrainte critique : une salle ne peut contenir qu'une seule fourmi par tour. Une option "visualizer" a été intégrée pour offrir une représentation graphique et animée du mouvement des fourmis.

Technologies utilisées

Le projet a été développé en Go (Golang), en utilisant exclusivement la bibliothèque standard pour la partie logique et algorithmique. Pour la partie graphique, la bibliothèque Ebitengine (Ebiten) a été exploitée afin de réaliser l'outil de visualisation.

Fonctionnalités principales

• Parsing et validation : Analyse syntaxique rigoureuse du fichier de données pour garantir un format compatible et conversion des informations en structures de données Go optimisées.

• Algorithme d'optimisation : Implémentation d'un algorithme de recherche de chemins maison, permettant de déterminer les trajectoires parallèles idéales pour minimiser le nombre de tours total. Les instructions de mouvement sont affichée en fin de programme dans le terminal.

• Visualisation dynamique : Intégration d'une fenêtre graphique permettant d'observer en temps réel le déplacement des fourmis à travers la colonie, facilitant ainsi la compréhension de la solution proposée.

Project Nightmare : Deadline Invaders

Game Design UI/UX Javascript

Vue d'ensemble

Project Nightmare : Deadline Invaders est un jeu web inspiré du classique Space Invaders, revisité avec une touche d'originalité et d'humour. Le jeu propose deux modes : le mode Histoire, qui intègre de courtes cinématiques pour contextualiser l'aventure et permet de nommer le "collègue relou", parfait pour se défouler après été témoin de conflits sur le lieu de travail, et le mode sans-fin, pour un peu plus de challenge. et le mode Sans-fin, axé sur le challenge et le score. Ce projet a été réalisé dans le cadre de ma formation au sein de l'école Zone01, en réponse à un cahier des charges technique rigoureux.

Technologies utilisées

Le projet a été développé exclusivement en HTML, CSS et JavaScript pur. Le cahier des charges interdisait strictement l'utilisation de l'élément Canvas ou de frameworks, imposant une gestion directe du DOM pour les animations. Les musiques sont libres de droits, tandis que le design des personnages et les cinématiques ont été intégralement créés par notre équipe.

Fonctionnalités principales

• Déplacements et tirs : Le joueur incarne un employé qui doit transmettre ses dossiers à ses collègues tout en évitant les interruptions du collègue difficile. Les mécaniques incluent le déplacement latéral et le "tir" de travail.

• Timer : En mode Histoire, le joueur dispose de 60 secondes pour accomplir ses objectifs et terminer le niveau.

• Power-Ups : Le joueur dispose d'une liste de Power-Ups prédéfinie, chacun présentant un avantage pour la complétion du niveau.

• Pause : Le joueur peut mettre le jeu en pause et le reprendre à tout moment.

Résultats

Les retours ont été extrêmement positifs. Les testeurs ont salué l'originalité du concept et l'aspect divertissant de la personnalisation de l'antagoniste. La sélection du nom à, en particulier, beaucoup inspiré nos testeurs. Cela prouve que le jeu répond à une demande existante !
